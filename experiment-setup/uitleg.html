Explanation for run

In this session, the sequences are visualized using a sliding window. New events (colored blocks) enter on the right side of the screen and vanish on the left side of the screen.

Interactions

If you left-click on a block, the block will get a blue border indicating that you have selected it as an anomaly.

If you right-click on a selected anomaly, the border of the block will be removed indicating that you have de-selected it as an anomaly. This could be useful if you have selected a block by accident or if you changed your decision.

Anomalies

Anomalies (deviations or 'afwijkingen' in Dutch) are present in almost all streaming sequences that you will evaluate. It is your task to detect these anomalies by clicking on the anomalous block.

Anomalies are single events that deviate in color from the contextual repetitive pattern.

Anomalous blocks replace the expected colored blocks in the sequence.

In case a pattern is changing, the first new block of a changing pattern is not considered an anomaly.

Click on the blocks that deviate in color or from the contextual repetitive pattern for that sequence (row). Whether you consider an event anomalous only depends on the pattern of the corresponding sequences (row). The first block of a changing pattern is not considered an anomaly.

In this session, the sequences are visualized using a compression and aggregation method. New events (colored blocks) enter on the right side of the screen and are compressed and, subsequently, aggregated once they get older.

In the compressed view, events are still shown in the same order as they entered the screen, but the width is reduced. The aggregation view shows a distribution of all the events that are older than the ones shown on in compressed and normal view. The separate colors indicate the percentage of events of a specific type with respect to the total number of events that are older than the events shown in compressed and normal view.

Anomaly Detection

An anomaly detector highlights potential anomalies by coloring the border of the event red. This algorithm is not perfect and can miss specific anomalies or misjudge and label normal events as anomalies. Be aware that you keep an eye on the unmarked events as well to detect anomalies.

Your interaction with the visualization will influence the sensitivity of the algorithm per sequence.

You can agree with the algorithm and left-click on the highlighted event to confirm the detection of an anomaly. The border color will change to blue.

You can disagree with the algorithm and right-click the highlighted event to un-mark the event, it is not registered as an anomaly.

It is important that you interact (agree/disagree) with the visualization to adjust the sensitivity of the algorithm. If you do not interact with a highlighted event, and it leaves the screen with a red border, the event is considered a normal event and not flagged anomalous.
