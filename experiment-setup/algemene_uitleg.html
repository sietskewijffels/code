Instructions

You are asked to detect anomalies in streaming event sequence data in four different visualization implementations. The order in which you will evaluate these implementations is randomized. A general explanation of the task is given below but will be made more specific based on the visualization implementation. After you have read and understood these instructions, you can continue to the explanation of the first session by clicking the button below.

Event sequences

The data that you are going to evaluate is visualized as colored blocks (events) that are organized in a sequence. The order of these colored blocks follows an easily recognizable repetitive pattern that is specific to that sequence but can change over time. Anomalies

Anomalies (deviations or 'afwijkingen' in Dutch) are present in almost all streaming sequences that you will evaluate. It is your task to detect these anomalies by clicking on the anomalous block.

Anomalies are single events that deviate in color from the contextual repetitive pattern.

Anomalous blocks replace the expected colored blocks in the sequence.

In case a pattern is changing, the first new block of a changing pattern is not considered an anomaly.

Basic setup

12 sequences of streaming data are shown stacked on each other at the same time. Every row represents a single sequence. The events (colored blocks) will stream into the screen from the right side of the screen to the left side of the screen and follow one of the patterns described above. The white spaces represent data gaps in time, they are not part of the pattern but are just small delays between the occurrence of events. You are monitoring the streaming data simultaneously and are asked to detect anomalies in the sequences by clicking on the anomalies. The events that are shown on the screen at the start of the session do not contain any anomalies.

Interactions

If you left-click on a block, the block will get a blue border indicating that you have selected it as an anomaly.

If you right-click on a selected anomaly, the border of the block will be removed indicating that you have de-selected it as an anomaly. This could be useful if you have selected a block by accident or if you changed your decision.

Summary

Click on the blocks that deviate in color or from the contextual repetitive pattern for that sequence (row). Whether you consider an event anomalous only depends on the pattern of the corresponding sequences (row). The first block of a changing pattern is not considered an anomaly.
